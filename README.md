# Dev_Ops_test

1. Добавлена автоматическая проверка форматирования кода и корректной работы приложения при запросе на слияние, push
2. Созданы тесты, проверяющие корректность работы функций: sum, sub, multi, power
3. Для реализации системы ветвлений в данном проекте была выбрана модель ветвления GitHub Flow

GitHub Flow — это легковесная модель работы с ветками (branching model), которая предназначена для организации рабочего процесса с использованием системы контроля версий, такой как Git, с акцентом на непрерывную поставку и обеспечение качества кода. Эта модель ветвления особенно хорошо подходит для небольших и средних проектов, где необходимо эффективно управлять изменениями и обеспечивать стабильность кодовой базы.
